Initiation à Claude Code · Module 1

CLAUDE.md, la mémoire de votre projet

Claude Code est un assistant IA qui travaille directement dans votre terminal. Mais à chaque nouvelle session, il repart de zéro. CLAUDE.md, c'est le fichier qui lui donne une mémoire durable.

~30 minutes Aucun prérequis technique
mon-projet — claude
# Un dossier vide, un terminal, c'est tout.
➜ mon-projet claude
✻ Bienvenue dans Claude Code

> /init

Le point de départ

Pourquoi Claude a besoin d'une mémoire

Chaque conversation avec Claude Code démarre avec une page blanche. Sans contexte, il doit redécouvrir votre projet, vos conventions et vos outils… à chaque fois.

🧠

Le problème

Claude ne se souvient de rien entre deux sessions. Répéter le contexte de son projet à chaque fois, c'est long et source d'erreurs.

📄

La solution

Un simple fichier texte nommé CLAUDE.md, placé à la racine du projet. Claude le lit automatiquement au démarrage de chaque session.

🎯

Le résultat

Claude connaît d'emblée votre projet, vos règles et vos commandes. Les réponses sont plus justes, plus cohérentes, dès le premier message.

La commande magique

/init : générer son premier CLAUDE.md

Pas besoin de partir d'une page blanche : la commande /init demande à Claude d'explorer le projet et de rédiger lui-même un premier CLAUDE.md. On le relit, on l'ajuste, et c'est parti.

1️⃣

Explorer

Claude parcourt les fichiers du dossier : code, configuration, documentation existante.

2️⃣

Synthétiser

Il en déduit la nature du projet, les technologies utilisées et les commandes importantes.

3️⃣

Rédiger

Il écrit un CLAUDE.md structuré qu'on peut ensuite éditer comme n'importe quel fichier texte.

🎬 DÉMO LIVE

On passe au partage d'écran

  1. Ouvrir un terminal dans un dossier vierge (ou un petit projet exemple).
  2. Lancer claude, puis taper /init.
  3. Observer ensemble le fichier CLAUDE.md généré : l'ouvrir, lire son contenu, le modifier en direct.
  4. Relancer une session et montrer que Claude « connaît » maintenant le projet.

Sous le capot

Anatomie d'un bon CLAUDE.md

C'est du Markdown tout simple. Ce qui compte, c'est ce qu'on y met — et surtout ce qu'on n'y met pas.

CLAUDE.md
# Projet : Portail RH

## Description
Application web interne de gestion
des demandes de congés.

## Commandes
- Lancer : npm run dev
- Tester : npm test
- Vérifier : npm run lint

## Conventions
- Interface en français
- Composants Vue en PascalCase
- Pas de librairie externe sans
  validation de l'équipe

## À ne pas faire
- Ne jamais modifier /legacy
- Ne pas toucher aux migrations
Description du projet

Deux ou trois phrases : c'est quoi, pour qui, avec quelles technologies. Assez pour orienter, pas plus.

Commandes essentielles

Comment lancer, tester, vérifier le projet. Claude s'en servira de lui-même au bon moment.

Conventions d'équipe

Les règles toujours vraies : style de code, langue, nommage. Ce qu'un nouveau collègue devrait savoir dès le jour 1.

Interdits et zones sensibles

Aussi important que le reste : dire à Claude ce qu'il ne doit jamais toucher évite bien des surprises.

L'art du dosage

Court, vrai, actionnable

Le piège classique : transformer CLAUDE.md en documentation exhaustive. Claude le relit à chaque session — chaque ligne doit mériter sa place.

✓ À faire

  • Rester court : viser moins d'une page à l'écran
  • Des instructions (« fais X »), pas des explications
  • Uniquement des informations toujours vraies
  • Mettre à jour quand le projet évolue
  • Le versionner avec le projet (Git) pour toute l'équipe

✕ À éviter

  • Copier-coller toute la documentation du projet
  • Des informations temporaires (« bug en cours sur… »)
  • Des détails que Claude peut découvrir seul dans le code
  • Des instructions contradictoires accumulées avec le temps
  • Le laisser vieillir sans jamais le relire

Pour aller plus loin

Une mémoire à plusieurs étages

Il peut exister plusieurs CLAUDE.md, et Claude les combine intelligemment, du plus général au plus spécifique.

~/.claude/CLAUDE.md
Niveau personnel (global)

Vos préférences à vous, valables dans tous vos projets : « réponds-moi en français », « explique tes choix », etc.

projet/CLAUDE.md
Niveau projet

Le fichier partagé avec l'équipe, à la racine. C'est celui que /init génère et celui qu'on utilise le plus.

projet/module/CLAUDE.md
Niveau sous-dossier

Pour les gros projets : des consignes spécifiques à une partie du code, chargées seulement quand Claude y travaille.

L'essentiel

À retenir du module 1

  1. CLAUDE.md = la mémoire du projet. Lu automatiquement au début de chaque session.
  2. /init pour démarrer. Claude rédige un premier brouillon, à vous de l'affiner.
  3. Court et toujours vrai. Des instructions, pas de la documentation.
  4. Un fichier d'équipe. Versionné avec le projet, il profite à tout le monde — humains inclus.
