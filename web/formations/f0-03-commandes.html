Les commandes : piloter Claude Code

Tout ce qui commence par / dans Claude Code est une commande. Certaines sont fournies d'origine, d'autres sont les vôtres. C'est le tableau de bord de vos sessions.

La carte du territoire

Deux familles de commandes

Taper / dans Claude Code affiche la liste complète, filtrable en tapant les premières lettres. Derrière ce menu, deux origines bien distinctes.

🏗️

Les commandes intégrées

Fournies avec Claude Code : gérer la session, le contexte, le modèle. Elles sont fixes et toujours disponibles — comme les boutons d'un tableau de bord.

🛠️

Les commandes personnalisées

Créées par vous ou votre équipe : vos raccourcis vers des instructions réutilisables. Depuis le module 2, vous savez déjà les créer — chaque skill devient une commande /nom !

Vos propres raccourcis

Créer une commande personnalisée

Une commande personnalisée, c'est un fichier Markdown dont le contenu est envoyé à Claude quand on tape /son-nom. Trois super-pouvoirs la rendent dynamique.

.claude/skills/revue/SKILL.md
---
name: revue
description: Revue de code des
  changements en cours
disable-model-invocation: true
---

## Changements à examiner
!`git diff`

## Normes de l'équipe
@normes-equipe.md

## Consigne
Fais la revue des changements
ci-dessus en te concentrant sur :
$ARGUMENTS

Classe tes retours par priorité.
$ARGUMENTS — les paramètres

Ce qu'on tape après la commande est injecté ici. /revue la sécurité → « …en te concentrant sur : la sécurité ».

@fichier — inclure un fichier

Le contenu du fichier référencé est ajouté automatiquement. Parfait pour joindre normes, gabarits ou configurations.

!`commande` — contexte dynamique

Exécute une commande du terminal et injecte son résultat. Ici, la commande reçoit les vrais changements du moment.

disable-model-invocation

Empêche le déclenchement automatique : la commande ne s'exécute que si vous la tapez. Utile pour les actions qu'on veut garder sous contrôle.

📜 Note pour les personnes qui connaissent déjà

Historiquement, les commandes vivaient dans .claude/commands/*.md. Ce format fonctionne toujours, mais il est désormais legacy : commandes et skills ont fusionné. Le format recommandé est .claude/skills/nom/SKILL.md — même invocation /nom, plus les fichiers d'appui et le déclenchement automatique en bonus.

🎬 DÉMO LIVE

On passe au partage d'écran

  1. Taper / dans Claude Code et explorer le menu des commandes ensemble.
  2. Dérouler 2-3 intégrées en direct : /context, /skills, /compact.
  3. Créer une commande personnalisée simple en direct (ou réutiliser le skill créé au module 2).
  4. L'invoquer avec un argument : /revue la lisibilité — et observer le résultat.

Le bon réflexe

Déclenchement automatique ou manuel ?

C'est le même fichier dans les deux cas — la vraie question est : qui décide du moment ? Claude, ou vous ?

AutomatiqueLaisser Claude décider

  • Tâches fréquentes où on veut que « ça marche tout seul »
  • La description du skill sert de déclencheur
  • Exemples : appliquer un gabarit, respecter une méthode de rédaction

ManuelGarder la main avec /

  • Actions ponctuelles ou sensibles qu'on veut déclencher soi-même
  • Ajouter disable-model-invocation: true au besoin
  • Exemples : revue de code, préparation de release, rapport hebdo

La boucle est bouclée

Les trois piliers, ensemble

CLAUDE.md, skills et commandes ne sont pas trois outils isolés : c'est un système. Le contexte, les savoir-faire, et le pilotage.

MODULE 01

CLAUDE.md

Le contexte permanent. Ce que Claude doit toujours savoir sur le projet. Généré avec /init, affiné à la main.

MODULE 02

Les skills

Les savoir-faire. Vos méthodes empaquetées, déclenchées automatiquement grâce à leur description — ou à la demande.

MODULE 03

Les commandes

Le pilotage. Gérer la session avec les intégrées, déclencher vos skills à volonté avec /nom.

L'essentiel

À retenir du module 3

  1. Taper / ouvre le menu. Le meilleur moyen de découvrir ce qui est disponible.
  2. Cinq intégrées à retenir : /init, /compact, /context, /skills, /diff.
  3. Trois super-pouvoirs : $ARGUMENTS, @fichier et !`commande` rendent vos commandes dynamiques.
  4. Commandes = skills. Un seul format à apprendre ; le choix, c'est automatique vs manuel.
